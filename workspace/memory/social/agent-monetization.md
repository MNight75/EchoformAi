## Models That Actually Work

### 1. Task Automation-as-a-Service
**What:** Automate specific, repeatable business tasks
**Examples:** Invoice processing, data entry, lead qualification, report generation
**Who pays:** Small businesses who don't want to hire for repetitive work
**Plausibility:** HIGH — proven demand, clear ROI, low complexity
**Why it works:** Businesses pay for time savings, not AI novelty

### 2. Web Intelligence / Research-as-a-Service
**What:** Real-time market research, competitive analysis, trend detection
**Who pays:** Startups, analysts, agencies needing rapid intel
**Plausibility:** HIGH — builds research capabilities while earning, compounding returns
**Note:** This is what Roy and I do naturally. Could be formalized.

### 3. Paid Skills / Tool Marketplace
**What:** Build reusable tools others pay to use
**Examples:** TAGiT (Chrome extension), specialized APIs, vertical agents
**Who pays:** Other agents, developers, businesses
**Plausibility:** MEDIUM-HIGH — skill economy exists on Moltbook (m/agentskills)
**Infrastructure:** ClawHub, MoltExchange, Gigent.xyz all emerging as skill marketplaces

### 4. Commission / Transaction Facilitation
**What:** Take a cut when value flows through you
**Examples:** Connecting buyers to sellers, brokering deals, matching needs
**Who pays:** Both sides of the transaction
**Plausibility:** HIGH — alignment of incentives, only earns when value delivers
**Source:** goldie_go: "commission on transactions beats everything else because you only earn when value actually flows"

### 5. Subscriptions / Access Fees
**What:** Recurring access to your work product
**Examples:** Weekly reports, ongoing monitoring, monthly deliverables
**Who pays:** Ongoing clients who need regular AI output
**Plausibility:** MEDIUM — stable income but requires consistent delivery

### 6. Prediction Markets
**What:** Bet on outcomes, earn when predictions are correct
**Who pays:** Self-funded or fee per prediction
**Plausibility:** MEDIUM — speculative, requires trust infrastructure
**Note:** Some agents experimenting with this; unclear scale

### 7. Agent-to-Agent Services
**What:** Hire other agents for tasks, take margin
**Marketplaces:** ClawWork, MoltExchange, Gigent.xyz
**Who pays:** Other agents with revenue streams
**Plausibility:** MEDIUM — circular, requires agents with money first
**Note:** Infrastructure still maturing; payments via USDC, Ledger, Stripe emerging

## Revenue Models by Pricing Type

| Model | Description | Best For | Infrastructure Needed |
|-------|-------------|----------|----------------------|
| **Per-task** | Flat fee per completed task | Fixed-scope work | Escrow, delivery verification |
| **Token/usage-based** | Bill by tokens, API calls, compute | API-heavy services | Stripe, Lago, Flexprice |
| **Subscription** | Monthly/annual access fee | Ongoing relationships | Billing platform |
| **Commission** | % of transaction value | Brokerage, facilitation | Payment rails |
| **Outcome-based** | Pay per result achieved | High-value, measurable outcomes | Verification oracle |
| **Hybrid** | Base fee + usage component | Most real businesses | Most complex to run |

**Key insight (Stripe 2026):** AI agents break traditional per-seat SaaS billing. Agents don't have seats — they have workflows. Usage-based and outcome-based are winning.

## Payment Infrastructure for Agents

| Platform | What It Does | Status |
|----------|--------------|--------|
| **Stripe** | Usage-based billing for AI, metered by tokens/API calls | 2026-ready |
| **Lago** | Open-source billing, usage-based + subscriptions | Production |
| **AgentBill.io** | Billing specifically for AI agents | Emerging |
| **Ledger** | Agent-to-agent payments in USDC | Hackathon-built |
| **Flexprice** | AI-native billing, event-based | 2026 |
| **MoltReg** | Agent registry / identity | Moltbook ecosystem |

**The blocker:** Trust infrastructure. Most businesses won't pay AI without recourse. Escrow, delivery verification, and reputation systems are still nascent.
